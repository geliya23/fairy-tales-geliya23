## Why
当前故事网站仅提供静态故事阅读功能，用户无法生成个性化故事。添加 AI 生成故事功能将允许用户通过 OpenAI 兼容的 API 创建自定义童话故事，提升用户体验和互动性。

## What Changes
- **新增 AI 设置面板**：允许用户配置 OpenAI 兼容 API 的 baseurl 和 apikey
- **新增模型选择功能**：通过 /models 端点获取可用模型列表，提供下拉选择
- **新增故事生成功能**：用户输入提示词，AI 生成完整童话故事
- **集成现有系统**：生成的故事保存为 .md 文件，并自动添加到故事列表
- **UI 增强**：
  - 头部添加"AI 生成故事"按钮
  - 弹出式设置面板（配置 API）
  - 故事生成对话框（输入提示词、选择模型、显示结果）
  - 生成进度指示器

## Impact
- **受影响的文件**：
  - `index.html`：新增 UI 组件和 JavaScript 逻辑
  - `stories.json`：需要动态更新以包含生成的故事
  - `story/` 目录：新增 AI 生成的故事文件
- **技术考量**：
  - 纯静态网站需要处理 CORS 限制（可能需要用户配置代理或使用支持 CORS 的 API）
  - API 密钥存储在 localStorage（需提醒用户安全风险）
  - 需要适配多个 OpenAI 兼容 API 提供商（如 OpenAI、Azure OpenAI、OpenRouter、Ollama 等）
- **安全影响**：
  - API 密钥仅存储在浏览器本地
  - 无服务端存储，降低密钥泄露风险
  - 提供清晰的安全使用说明

## 关键设计决策
- 使用 fetch API 直接调用 OpenAI 兼容端点
- 支持流式和非流式响应（优先流式体验）
- 生成的故事遵循现有 Markdown 格式（第一行为标题，第二行为副标题）
- 允许用户手动编辑生成的故事再保存
