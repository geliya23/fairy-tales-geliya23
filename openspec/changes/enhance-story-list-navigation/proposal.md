## 为什么
当前的故事列表在故事数量增加时（如10个以上）会变得冗长，用户需要滚动大量内容才能找到目标故事。这种线性列表设计缺乏有效的导航和筛选机制，严重影响了用户体验。随着AI生成故事的不断添加，问题会更加突出。

## 什么改变
### 添加的功能
1. **搜索框** - 在故事列表顶部添加实时搜索框，允许用户通过标题快速定位故事
2. **故事分组** - 将故事按来源分为两组：
   - 原创故事（手动创建的故事文件）
   - AI生成故事（通过AI生成的故事文件）
3. **折叠/展开功能** - 每个分组可以独立折叠或展开，默认展开原创故事，折叠AI生成故事
4. **计数徽章** - 在每个分组标题旁显示该组包含的故事数量

### 用户界面改进
- 重新设计左侧故事列表区域，添加搜索和分组控件
- 优化列表项的显示密度和视觉层次
- 改善交互动画和状态反馈

## 影响
- **受影响的规范：** 需要修改现有故事列表显示逻辑
- **受影响的代码：** `index.html` 中的故事列表渲染部分，JavaScript中的故事加载和管理逻辑
- **向后兼容：** 完全向后兼容，不破坏现有功能
- **性能影响：** 搜索功能为客户端过滤，无性能问题

## 非目标
- 移动端响应式适配（将在后续变更中处理）
- 故事分类标签系统（超越原始/AI生成的基础分组）
- 复杂的多维度筛选
